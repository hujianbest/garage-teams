# 作业合同模板

当项目需要把逻辑工件与团队现有交付物做显式映射时，可使用此模板。

这个模板的目标不是增加流程负担，而是把 `mdc` 路由真正依赖的最小工件和批准规则固定下来。

```markdown
# 作业合同

## 项目

- 名称: <项目名称>
- 状态: 草稿 | 进行中
- 负责人: <团队或责任人>

## 工件映射

| 逻辑工件 | 实际路径 | 批准信号 | 是否必需 |
|---|---|---|---|
| 需求规格 | `docs/specs/<file>.md` | `状态: 已批准`，兼容 `Status: Approved`，且已完成真人确认 | 是 |
| 设计文档 | `docs/designs/<file>.md` | `状态: 已批准`，兼容 `Status: Approved`，且已完成真人确认 | 是 |
| 任务计划 | `docs/tasks/<file>.md` | `状态: 已批准`，兼容 `Status: Approved` 或 tasks-review `通过` / `PASS` | 是 |
| 进度记录 | `task-progress.md` | 无 | 是 |
| 发布说明 | `RELEASE_NOTES.md` | 无 | 建议 |
| 评审记录 | `docs/reviews/` | `通过` / `需修改` / `阻塞`，兼容 `PASS` / `REVISE` / `BLOCKED` | 建议 |
| 验证记录 | `docs/verification/` | 命令输出摘要或验证结论 | 建议 |

## 最小必需工件

以下工件构成 `mdc` 的最小契约。若项目没有自己的等价映射，默认应至少具备这些路径中的对应物：

- 需求规格
- 设计文档
- 任务计划
- `task-progress.md`

若这些工件缺失，`mdc-workflow-starter` 应按更上游阶段处理，而不是默认继续实现。

## 建议工件

以下工件不是所有项目都必须一开始具备，但建议逐步补齐：

- `RELEASE_NOTES.md`
- `docs/reviews/`
- `docs/verification/`

这些工件一旦存在，就应被视为更高优先级的阶段证据来源。

## 路由证据

| 用途 | 路径 | 说明 |
|---|---|---|
| 进度 / 当前状态 | `task-progress.md` 或等价文件 | 主路由证据来源 |
| 评审记录 | `docs/reviews/` | 阶段批准证据 |
| 验证记录 | `docs/verification/` | 回归 / 完成验证证据 |
| 发布说明 | `RELEASE_NOTES.md` | 面向用户的完成证据 |

## 真人确认记录

对需求规格和设计文档而言，仅有评审通过还不够；还必须有真人确认已经完成的证据。

建议至少在以下位置之一留下记录：

- 工件本身的状态字段或确认章节
- `docs/reviews/` 中的 review 记录
- `task-progress.md` 中的阶段状态记录

如果三者都没有，`mdc-workflow-starter` 应按“尚未批准”处理。

## 阶段规则

1. 没有已批准需求规格前，不进入设计
2. 没有已批准设计前，不进入任务规划
3. 没有已批准任务计划前，不进入实现
4. 没有回归门禁和完成门禁证据前，不宣称完成

## 什么叫“已批准”

在本项目中，可将以下信号视为批准证据：

- `状态: 已批准`
- 兼容旧写法：`Status: Approved`
- 带有 `通过` 结论的评审记录
- 兼容旧写法：带有 `PASS` 结论的评审记录
- 进度或验证记录中的阶段标记

对规格和设计而言，还应有真人确认已经完成的证据；若缺失，则“评审通过”不等于“已批准”。

- 在这里记录工件命名偏差
```

## 使用说明

- 合同应尽量简短、稳定
- 仅在工件路径或批准规则变化时更新
- 优先映射现有文档，而不是引入重复交付物
- 优先固定最小契约，再逐步扩展建议工件

## 示例：映射已有项目文档

如果项目已经有自己的文档命名方式，可以用下面这种方式映射，而不强制改名：

```markdown
| 逻辑工件 | 实际路径 | 批准信号 | 是否必需 |
|---|---|---|---|
| 需求规格 | `docs/prd/release-approval-prd.md` | `状态: 已批准` + 评审记录中已完成真人确认 | 是 |
| 设计文档 | `docs/architecture/release-approval-hld.md` | `Status: Approved` | 是 |
| 任务计划 | `docs/project-plans/release-approval-plan.md` | `通过` | 是 |
| 进度记录 | `task-progress.md` | 无 | 是 |
```
