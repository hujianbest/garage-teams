# Garage 用户契约

## 适用对象

这份契约面向 Garage 的核心用户——solo creator：一个人扛产品、技术、内容、运营的独立创作者。你不是企业的螺丝钉，你是自己的全部。

## 我们承诺什么

### 1. 能力不丢失

你今天积累的每一个 skill、每一条知识、每一次经验，明天还在。

- 所有数据存储在你的 git 仓库里，不是我们的服务器上
- git 提交历史就是你的能力成长时间线
- 即使 Garage 项目本身停止维护，你的数据仍然完整可读

### 2. 宿主自由

你可以换工具，但不能因为换工具就丢能力。

- 核心约定不包含任何单一宿主的特定术语
- 换宿主只需要换适配层，不需要重新积累 skills 和知识
- 我们优先支持 Claude Code，但不会为此牺牲可迁移性

### 3. 渐进增强

第一天就能用，用得越多越强。不存在"先配置 50 个东西才能开始"的门槛。

- 零配置启动：clone 仓库，开始使用
- 每一层复杂度都有明确触发条件，不会突然变复杂
- 简单场景不需要理解高级功能

### 4. 透明可审计

你随时可以知道系统知道什么、做了什么、为什么这么做。

- 知识条目带来源锚点——每条知识都能追溯到产生它的场景
- 设计决策带理由——每个约定都能回答"为什么是这样的"
- Agent 的行为可回溯——不搞黑箱自动化

### 5. 你做主

系统提供建议和自动化，但方向盘在你手里。

- 关键决策（架构选择、优先级、范围变更）永远由你确认
- 系统可以自动积累知识和识别模式，但不能替你发布、部署或删除
- 所有自动化都有开关，没有不可关闭的自动行为

## 我们不承诺什么

### 不承诺 100% 自动化

Garage 追求的是"人机共生"，不是"完全自动"。有些事情机器做得好（知识索引、模式识别），有些事情只有人能做（价值判断、创意决策）。我们不会为了"看起来更 AI"而把人的判断力排除在外。

### 不承诺多用户协作

当前阶段，Garage 为一个人设计。多用户协作是未来可能的方向，但不是现在的承诺。如果你需要团队使用，可以在 Garage 之上自行封装。

### 不承诺向后兼容到永远

我们承诺渐进演进，不承诺"所有旧格式永远支持"。当格式需要变更时，我们会提供迁移路径和迁移工具，但不会为了兼容性而阻止合理的演进。

### 不承诺性能极致

Garage 是面向 solo creator 的本地工具，不是企业级系统。知识库条目在百级、千级时表现良好，但我们不会为了支持百万级而引入数据库和复杂基础设施——至少在可预见的未来不会。

## 你的责任

契约是双向的。为了让 Garage 更好地服务你：

1. **定期提交 git**：你的提交历史就是你的能力备份。不提交，不积累。
2. **review 系统的建议**：系统积累的知识和识别的模式需要你偶尔检视，避免错误知识的扩散。
3. **接受渐进式成长**：不要追求一步到位。先用起来，让系统在你的使用中自然成长。
4. **贡献回流**：如果你积累了高质量的 skills 或知识模式，考虑分享给社区。一个人的产出可以惠及很多人。

## 契约的变更

这份契约是活的。当 Garage 的定位、能力或边界发生重大变化时，这份契约会同步更新，并保留变更历史。

你可以在 `docs/soul/` 下找到这份契约以及 Garage 的其他灵魂文档。
