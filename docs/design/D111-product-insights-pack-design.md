# D111: Product Insights Pack Design

- Design ID: `D111`
- 状态: 草稿
- 日期: 2026-04-11
- 定位: 展开 `Product Insights` 作为 `Garage Team` 能力面的具体设计。
- 关联文档:
  - `docs/design/D11-reference-pack-designs.md`
  - `docs/features/F15-pack-platform-and-collaboration.md`
  - `docs/features/F14-continuity-and-growth.md`

## 1. 设计目标

- 定义 product insights agents / roles / outputs
- 说明它如何与长期 continuity 和 evidence 主线结合

## 2. 主要工作对象

`Product Insights` pack 主要处理：

- research inputs
- problem framing
- insight outputs
- bridge-ready handoff artifacts

## 3. 最小角色面

至少应存在这些 team-facing roles：

- 研究角色
- 结构化角色
- handoff / bridge-ready 角色

## 4. 最小输出面

至少应产生可被下游消费的：

- insight artifacts
- scoped problem statements
- bridge-ready outputs for coding or other packs
- supporting evidence records

## 5. continuity 与 evidence 结合

- 研究过程的关键判断进入 evidence
- 可复用研究方法和结构化方式可进入 continuity 候选
- pack 不直接决定 memory / skill promotion，而是把材料送进 shared growth 主线

## 6. 非目标

- 不把 Product Insights 设计成通用聊天代理
- 不让它自己拥有 bridge acceptance 真相
- 不让它越权定义 execution/provider 规则

## 7. 设计完成标准

- role / output / evidence hooks 清楚
- 与 shared growth 和 bridge 主线的关系清楚
- 下游 implementation task 不再需要猜 pack 最小能力面
