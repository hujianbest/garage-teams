# garage-agent v0.1.0 — First public release

`garage-agent` is a local-first home for agent skills, knowledge, and experience, built for solo creators who want agent capability that starts small, grows with real work, and stays portable across hosts (Claude Code, Cursor, OpenCode).

This is the first public release. It packages 14 closed delivery cycles (F001–F014) into a real open-source artifact: legal, community, security, CI, and PyPI metadata are all in place.

## What's in this release

### Capability (carried from F001–F014, unchanged in v0.1)

- **`garage` CLI** — `init` / `status` / `run` / `recommend` / `sync` / `session import` / `memory review`; knowledge sub-commands (`add` / `edit` / `show` / `delete` / `search` / `list` / `link` / `graph` / `export`); experience sub-commands; pack lifecycle (`install` / `ls` / `uninstall` / `update` / `publish`); skill mining (`skill suggest` / `skill promote`); workflow recall (`recall workflow`)
- **3 first-class host adapters** — Claude Code, Cursor, OpenCode (project- or user-scoped)
- **4 distributable packs** — `garage` (3 skills + 3 production agents), `coding` (24 HF workflow skills), `writing` (5 skills), `search` (1 skill)
- **File-first runtime** — sessions, knowledge entries, experience records, host installer manifest, sync manifest all under `.garage/` as readable files
- **Memory flywheel** — `garage sync` pushes top-N memory into each host's context surface; `garage session import --from <host>` pulls host conversation history back into Garage for extraction
- **Pack lifecycle** — install, update, publish (with sensitive-pattern scan + force-push prompt), uninstall, plus anonymized knowledge export

## Install

```bash
git clone https://github.com/hujianbest/garage-agent.git
cd garage-agent
uv pip install -e .   # or: poetry install
```

PyPI release is prepared but not yet published to the registry. The release artifacts (`garage_agent-0.1.0-py3-none-any.whl` and `garage_agent-0.1.0.tar.gz`) are attached to this release; once the maintainer runs `twine upload`, `pip install garage-agent` will be available.

## Quick start

```bash
# materialize garage / coding / writing / search packs into your host directories
garage init --hosts claude,cursor --yes

# inspect state
garage status

# push top-N memory into each host's context surface
garage sync --hosts claude,cursor

# pull host conversation history back into garage
garage session import --from claude --all
```

## Known limitations (carried to v0.2+)

- `ruff` (506) + `mypy` (73) findings need a dedicated cleanup cycle before going blocking in CI
- Cursor history reader is a stub (deferred to D-1010)
- HOST_REGISTRY plugin path requires source edit for a 4th host (D-705)
- Pack signature / GPG (D-1212) and `pack search` / central registry (D-1214) are Stage 4 work for future cycles
- Real 3-way merge for `pack update --preserve-local-edits` (D-1211); current behavior is warn-then-overwrite
- Cross-machine `garage sync` still relies on manual `git push` (D-707 carry-forward)
