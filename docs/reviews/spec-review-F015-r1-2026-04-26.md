# Spec Review — F015 r1 (Agent Compose)

- 日期: 2026-04-26
- 审阅人: hf-spec-review reviewer subagent (Cursor Agent task)
- 范围: `docs/features/F015-agent-compose.md` (commit `4b59555`; reviewed against current tree)

## Verdict: CHANGES_REQUESTED

Core scope and traceability to F013-A / KnowledgeStore are mostly sound, but the spec over-generalizes "F011 agent schema" to three files when one is intentionally a no-frontmatter sample, and **Acceptance 8.4 contradicts INV-F15-5** (overwrite of a listed production agent). These need resolution before the spec is a stable hf-design input.

## Findings (critical → important → minor → nit)

### Critical / USER-INPUT

**Cr-1 — BDD 8.4 vs INV-F15-5 直接矛盾**

- **Spec**: §2.1 INV-F15-5 (L125–127, L196) — `code-review-agent.md` / `blog-writing-agent.md` / `garage-sample-agent.md` byte 不变; 但 §8.4 (L278–286) 要求对**已存在的** `packs/garage/agents/code-review-agent.md` 执行 `compose` 并 y → overwrite.
- **冲突**: 任何覆盖都会改变文件字节, 与 INV-F15-5 的 sentinel 不兼容.
- **r2 需作者裁定 (USER-INPUT)**: 二选一 (a) 8.4 改为对**非 F011 文件名**的覆盖场景 (例 `code-review-agent-v2`); 或 (b) 收窄 INV-F15-5 的保护集 (仅 .md byte invariant 但不限制路径覆盖); 或 (c) 8.4 标为 "out-of-tree 副本" / `--target-pack` 切走.
- **auto mode 推荐**: 选 (a) — BDD 8.4 改 agent 名为 "demo-overwrite-agent" 而非 "code-review-agent"; 这样 INV-F15-5 既守 F011 既有 3 个 agent byte, BDD 又能覆盖 overwrite prompt 路径.

**Cr-2 (LLM-FIXABLE) — "F011 3 个 production agent" 作唯一 schema 参考与 `garage-sample-agent` 实码不符**

- **Spec**: L17, L24, L78, L196, HYP-1502 (L213) 等将三者统一为 frontmatter + "When to Use" / "How It Composes" / "Workflow" 模板.
- **实码** `packs/garage/agents/garage-sample-agent.md`: **无 YAML front matter**; 章节为 `## 行为` / `## Notes`, **无** "When to Use" / "How It Composes" / "Workflow" (L1-L11). README 也写明故意无 front matter (FR-708 容错路径).
- **影响**: CON-1504, FR-1501 draft 结构, "strict 遵 F011 既有" 表述若包含 `garage-sample-agent`, 与事实冲突.
- **r2 修订**: 显式收窄 "F011 compose 模板参考" 子集为 **2 个 agent** (`code-review-agent.md` + `blog-writing-agent.md`); `garage-sample-agent.md` 单独说明为安装/容错样本, 不进 CON-1504 的逐字结构.

### Important

**Im-1 (LLM-FIXABLE) — FR-1501 Edge vs FR-1503 / §8.3 missing skill 行为不一致**

- **FR-1501** (L140-142): missing skills → `missing_skills` + draft 用 placeholder
- **FR-1503** (L169) 与 §8.3 (L271-276): exit 1 且不创建 `x.md`
- **r2 决策**: auto mode 锁定 — CLI 层 (FR-1503) 严格: missing skill → exit 1 不写盘; library 层 (FR-1501 ComposeResult.missing_skills) 仍返回 missing skills 用于诊断 (但 CLI 不消费成功路径). 即 library 是 "返回 missing 列表 + 仍生成 draft" 但 CLI 在 missing > 0 时直接 exit 1 不写, library 仍可被未来其他调用方按需用部分 draft.

**Im-2 (LLM-FIXABLE) — RSK-1501 / FR-1505 "last compose <ts>" 未闭合**

- **§2.1 E** (L107) 与 FR-1505 表 (L184) 只承诺 "garage has N agents"; L107 提纲含 "last compose: <ts>", 但 FR-1505 表体未要求时间戳.
- **r2 修订**: cache.json 是 optional (L118); "last compose" 时间戳从 spec 删除 (避免依赖 cache.json); FR-1505 显示 "Agent compose: <pack> has N agents" — 不带 ts 后缀.
