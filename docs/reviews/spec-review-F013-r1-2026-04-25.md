## Findings (排序: critical → important → minor → nit)

### Cr-1 (critical, LLM-FIXABLE) — 错误的 orchestrator 类名

- **位置**: 关联列表 L8、L20; 图 L36; 2.1 A1 L76; FR-1301 L144; 8.1 L250; 9 T4 L318。
- **问题**: 全文使用 `ExtractionOrchestrator`, 代码中为 `MemoryExtractionOrchestrator`（`src/garage_os/memory/extraction_orchestrator.py`）。
- **r2**: 全局替换为 `MemoryExtractionOrchestrator`, 并在 hook 处写清**调用点**（例如 `SessionManager` 在归档/提取链末尾 vs orchestrator 方法末尾），避免与 CON-1301 对 "F003 API" 的表述冲突（见 Im-2）。

### Cr-2 (critical, LLM-FIXABLE) — `garage run` 的 CLI 形态错误

- **位置**: L103、L275–276、FR-1304 L179。
- **问题**: 规格写 `garage run hf-test-driven-dev --skill <name>`. 实际 `garage run` 仅接受**位置参数** skill 名, 无 `--skill`（见 `src/garage_os/cli.py` 约 L2070–2078: `run_parser.add_argument("skill_name", ...)`）。
- **r2**: 改为例如 `garage run <skill_name>` 或 `garage run verdict-format`（与 skill 目录名/skill id 一致）; 在 FR-1304 中删除 `--skill` 及错误示例字符串。

### Cr-3 (critical, LLM-FIXABLE + USER-INPUT) — `ExperienceRecord` 无 `phase` 字段; `KnowledgeEntry` 无 `problem_domain` 字段

- **位置**: FR-1303 L92–95（phase）; FR-1301/聚类/8.1 多处 `problem_domain`（L76、L146、L245 等）.
- **证据**: `ExperienceRecord`（`src/garage_os/types/__init__.py` L125–147）无 `phase`. `KnowledgeEntry`（L102–121）无 `problem_domain`, 仅 `front_matter` 可存额外键（`knowledge_store._front_matter_to_entry` 把整份 fm 存进 `front_matter`）。
- **r2 (LLM-FIXABLE)**: FR-1303 Workflow 骨架来源改为**已有字段**（如 `task_type` / `lessons_learned` / `key_patterns`）或明确 "从 `source_evidence_anchors` / 扩展字段读取（若 F013 扩 schema 则违 CON-1301，禁止）"。
- **r2 (USER-INPUT)**: 对 Knowledge 侧 **problem_domain** 必须写死规则: 只认 `entry.front_matter.get("problem_domain")`, 或映射 `topic` / tag, 否则 8.1 Given 与 FR-1301 聚类无法落地。

### Cr-4 (critical, LLM-FIXABLE) — INV-F13-1 与 A3/FR-1303 矛盾

- **位置**: INV-F13-1 L203; A3 L96–97; FR-1303 L170。
- **问题**: INV 称除 `.garage/skill-suggestions/` 外不向 "用户拥有 path" 写文件; A3 又要求模板写到**临时目录**（典型为 `/tmp` 或 `tempfile`）— 在字面上违 INV。
- **r2**: 二选一写清: (a) 允许写到系统 temp 并改 INV 例外; 或 (b) 草稿仅存在 `.garage/skill-suggestions/.../draft` 下, promote 再拷到 `packs/`.

### Im-1 (important, LLM-FIXABLE) — `ExperienceIndex.search` 引用签名错误

- **位置**: L21。
- **问题**: 规格写 `search(skill_ids, key_patterns, tags)`. 实现为 `search(task_type=None, skill_ids=None, domain=None, key_patterns=None)`（`experience_index.py` L77–84）, 无 `tags` 参数。
- **r2**: 改为与实际一致; 与 tag 相关过滤需指向 `key_patterns` 或别的方式并一句话说明。

### Im-2 (important, LLM-FIXABLE) — CON-1301 "字节级不变" 与 "orchestrator hook" 的张力

- **位置**: CON-1301 L213; INV-F13-5 L207; FR-1301 L144。
- **问题**: 在 `MemoryExtractionOrchestrator` 内增加 hook 会改 `F003` 源码; 若仅 "数据结构不变" 应写清 **允许新增可选 callback / 非 breaking 扩展**, 避免审阅/实现时按字面 "字节级" 卡死。

### Im-3 (important, LLM-FIXABLE) — `commit_sha` / `test_count` 无稳定数据源

- **位置**: FR-1303 L95, L172。
- **问题**: 标准 `KnowledgeEntry` / `ExperienceRecord` 无这两个字段; 仅可能存在于 `front_matter` 或 evidence anchor 的 ad-hoc 结构里。
- **r2**: 写 "从 `source_evidence_anchor` / `source_evidence_anchors` 按 schema 取可选字段, 缺省则 placeholder", 并给出 1 条 BDD 用 mock front matter 锚定.

### Im-4 (important, LLM-FIXABLE) — skill-anatomy 符合性表述不全

- **位置**: FR-1303 L90–95, L171; INV-F13-4 L206。
- **问题**: `docs/principles/skill-anatomy.md` 除 Workflow/Contract 外还强调 **When to Use**、**边界 (不适用)**、**目录 anatomy** 等; 规格以 "5 段/7 原则" 混用, 未显式把 "When to Use / 与相邻 skill 区别" 纳入模板生成必选段。
- **r2**: 列出生成器必须产出的**章节清单**与 skill-anatomy 各条原则的对照表（可引用原则编号而非含糊 "5 段"）.

### Im-5 (important, LLM-FIXABLE) — FR-1302 `--threshold` 语义在全文不一致

- **位置**: A1 L86 "仅本次重扫" vs L161 "本次 list 用"; FR-1301 L160 `--rescan`。
- **问题**: `--threshold` 是只影响**列表/评分展示**, 还是**触发/重算聚类**（与 rescan 组合）, 易歧义.
- **r2**: 用表格固定: `--threshold` + `suggest` 默认; `--rescan` + `--threshold` 时是否全量重扫并写新 proposal.

### Im-6 (important, LLM-FIXABLE) — RSK-1301 与 A5/FR-1305 显示规则冲突

- **位置**: RSK-1301 L233; FR-1305 L193; A5 L111–112。
- **问题**: RSK 要求即使无 proposal 也显示 "Skill mining: scanned X/Y/Z"; A5/FR-1305 说 💡 行 **proposed>0 才显**（且 Z=0 的摘要 elsewhere）.
- **r2**: 合并为单条 `garage status` 规约: 元数据行（scan 计数）是否始终输出; 💡 行是否仅在 proposed>0; 与 RSK-1301 的 "永不静默" 一致。
