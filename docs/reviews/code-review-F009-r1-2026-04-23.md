**I-3**: **`UserHomeNotFoundError` 是死路径** [`important`][`LLM-FIXABLE`][`CR3`/`CA1`]
- 锚点: `manifest.py:48`(类型定义); `pipeline.py:312-314 / 340-345` (调用 `Path.home()` 但**无 try/except**); `cli.py:303-305` (catch 路径)
- spec 期望: ASM-903 + ADR-D9-10 "pipeline 在 user scope 路径解析时若 `Path.home()` 抛 `RuntimeError`, exit 1 + stderr 'Cannot determine user home directory: ...'"
- 实际: `Path.home()` 在 adapter / pipeline 多处直接调用, 无任何 `try ... raise UserHomeNotFoundError(...) from e` 转译
- 影响: 真实生产环境 `$HOME` 未设置时, 用户看到 Python `RuntimeError` traceback 而非设计承诺的 stderr + exit 1
- **Decision**: executor 在 cycle 内修复 + 加测试 (真实代码缺陷, 不是仅测试守门弱)

**I-4**: **`_merge_with_existing` 仍用 2 元组 key 与 phase 4 5 元组 key 不对称** [`important`][`LLM-FIXABLE`][`CR2`/`CA4`]
- 锚点: `pipeline.py:430` (`fresh_keys = {(e.src, e.dst) for e in fresh.files}`); `pipeline.py:435` (`if e.host not in fresh_hosts and (e.src, e.dst) not in fresh_keys`)
- 期望: 与 phase 3 / phase 4 5 元组 key 对称
- 影响: 用户先 `garage init --hosts claude --scope user`, 再 `garage init --hosts claude --scope project` 时, 第二次 init 的 `_merge_with_existing` 会 drop 所有 prior user scope claude entry — manifest 丢失 user scope 安装记录, 但磁盘上 `~/.claude/skills/` 文件还在, 形成 manifest 与磁盘漂移
- **Decision**: executor 在 cycle 内修复 + 加测试 (真实代码缺陷, 跨 scope 真实漂移风险)
