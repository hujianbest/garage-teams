## Findings (critical → important → minor → nit)

### Critical / USER-INPUT

**Cr-1 — `--yes` 与现有 `garage init --yes` 语义冲突 (FR-1603 / INV-F16-3 / CON-1604)**

- **Spec**: §2.1 C / FR-1603 / BDD 要求 `garage init --yes` 跳过 memory prompt 且 enable extraction; §1.4 称 "`--yes` 显式 opt-in"
- **Code**: `init` 的 `--yes` 仅跳过 **host 安装**交互; 无 `--hosts` 时等价 `--hosts none`, 返回空 host list (`_resolve_init_hosts`, `cli.py:327-364`, 尤其 L362-363); `DEFAULT_PLATFORM_CONFIG` 仍 `extraction_enabled: False` (L171-183)
- **结论**: 若按 spec 实现, F016 的 `garage init --yes` → enable memory **直接冲突** F007 既有 "`--yes` = CI 非交互不装 host"; INV-F16-3 "现有 `garage init --yes` 不破" 不成立
- **r2 USER-INPUT 决策**: 二选一 (a) F016 只用 `--no-memory` flag, **不让 `--yes` 改 memory state** — 默认 enable=true 仅在 interactive prompt 路径; non-TTY 默认 disable (与既有 init 行为一致); 或 (b) 加新 flag `--memory` / `--enable-memory` opt-in, 不重载 `--yes`
- **auto mode 推荐**: 选 (a) — 默认 init **不 enable memory** (与既有 `--yes` 行为兼容); enable 路径仅 (1) interactive prompt y; (2) 显式 `garage memory enable`

**Cr-2 — HYP-1605 与 F003 实现不符**

- **Spec** §6 HYP-1605: "archive 时无 LLM 则 extraction no-op"
- **Code**: extraction_enabled=true 时, `MemoryExtractionOrchestrator._generate_candidates` **本地** 生成 candidate, 不调 LLM (`extraction_orchestrator.py:194-280`)
- **r2 USER-INPUT 修订**: HYP-1605 改写为 "archive_session 触发 extraction 是本地 signal-driven 候选生成 (无 LLM 调用); F016 enable 后用户在 host 内的对话仍通过 F010 `garage session import` 显式 ingest, 与 F003 默认行为一致"
