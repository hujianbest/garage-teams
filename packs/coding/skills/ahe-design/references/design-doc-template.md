# 设计文档模板

若 `AGENTS.md` 为当前项目声明了设计模板，优先使用项目约定。以下为默认结构。

## 默认结构

```markdown
# <主题> 实现设计

- 状态: 草稿
- 主题: <主题>

## 1. 概述
## 2. 设计驱动因素
## 3. 需求覆盖与追溯
## 4. 架构模式选择
## 5. 候选方案
## 6. 选定方案与关键决策
## 7. 架构视图
## 8. 模块职责与边界
## 9. 数据流、控制流与关键交互
## 10. 接口与契约
## 11. 非功能需求与约束落地
## 12. 失败模式与韧性策略
## 13. 测试与验证策略
## 14. 关键决策记录（ADR 摘要）
## 15. 明确排除与延后项
## 16. 风险、待定问题与任务规划准备度
```

## 编写要求

- 区分需求与设计
- 清晰定义边界和职责
- 用追溯视角说明关键需求被哪些模块、流程或接口承接
- 解释为什么所选方案适合该规格
- 说明设计怎样支撑后续任务规划，但不要写成任务计划
- 提供足够支撑任务拆解的细节，但不要写成逐行编码说明

## 保存路径

默认：`docs/designs/YYYY-MM-DD-<topic>-design.md`

若 `AGENTS.md` 声明了设计路径映射，优先使用映射路径。
