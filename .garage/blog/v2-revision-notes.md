# 博客 V2 修改要点

基于用户反馈，对初稿进行以下修改：

## 1. 第一章摄影网站案例：降调

- 不要夸张描述"Bug满天飞""改一处崩三处"
- 实际情况：改完之后确实不知道哪里还有 Bug，需要反复测试才能发现问题，效率并不比人做高多少
- 也没有"一两天就搞完"这么夸张

## 2. 需求分析部分：标注为借鉴，非本人核心

- 用户核心强项是软件设计（DDD、抽象建模），第一篇文章已体现
- 需求分析（EARS、BDD、MoSCoW）不是用户最熟悉的领域，是借鉴来的
- 在文章中明确声明这是借鉴的方法

## 3. 第五章 Harness Flow：聚焦方法论而非功能

- 重点不是介绍框架功能，而是讲里面采用的软件工程方法论
- 强调这是一个有方法论支撑的工作流，不是随意发挥的工作流
- 对标 README 中的 Methodology 层表格

## 4. 全文统一用摄影做类比

- 第2章邮件类比 → 改为摄影系统类比（如做一个摄影作品管理平台）
- 第4章讲"作品风格、审美"→ 用摄影的构图、光影、后期风格来类比
- AI 带来的生产力 vs 摄影的艺术性，形成鲜明对比

## 5. 预留配图位置

- 在关键位置插入 `<!-- 插图: 描述 -->` 占位符
- 如模型能生成配图则生成，否则留空让用户后续插入

## 6. 增加与第一篇文章的互动

- 引用第一篇的核心概念：抽象、分治、权衡
- 展示这些概念在 AI 时代如何被重新激活

## 7. 增加与书籍观点的互动

- 引用《代码整洁之道》《重构》《Google 软件工程》等书中的具体观点
- 展示这些书的智慧在 AI Coding 语境下的新意义
