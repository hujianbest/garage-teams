# 独立版需求澄清评测

这个目录包含独立版 `mdc-specify` 的评测 prompts。

## 目的

这些评测用于验证该 skill 是否真正做到：

- 先澄清需求，再起草规范文档
- 借鉴 `brainstorming` 的一问一答推进方式
- 不提前进入设计或实现
- 输出规范草稿后做轻量自检
- 明确给出“可进入设计”或“仍需继续澄清”的判断

## 建议评分关注点

1. 是否优先澄清目标、范围、非目标、约束与验收标准
2. 是否在需求和设计决策之间做出清晰区分
3. 是否在范围存在歧义时给出可选切分方案
4. 是否产出结构清晰的规范文档草稿
5. 是否在结束时执行轻量自检并给出 readiness 判断
