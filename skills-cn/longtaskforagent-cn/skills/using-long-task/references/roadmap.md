# 路线图 — 未来增强

## P2：并行 Agent 调度（R8）

**状态**：已文档化，尚未实现

**目标**：支持独立特性实现的并行调度。

**需求**：
- R8.1：识别无相互依赖、可并行执行的特性
- R8.2：使用 Task 工具并行派发 implementer 子代理
- R8.3：合并结果、检测冲突、并行完成后运行完整测试套件

**设计说明**：
- 仅对真正独立的任务并行（无共享文件、无依赖）
- 并行完成后始终运行完整测试套件
- 若发现冲突，改为顺序解决
- 参见 `subagent-development.md` 中的并行调度小节

## P2：插件发现系统（R12）

## P2：自动更新机制（R13）

**状态**：已文档化，尚未实现

**目标**：在技能加载时检查新版本。

**需求**：
- R13.1：在 SKILL.md frontmatter 或 VERSION 文件中增加版本号
- R13.2：技能加载时检查远程仓库是否有更新版本
- R13.3：若有更新则通知用户（绝不自动更新）

**设计说明**：
- 基于 Git 的检查：对 origin 执行 `git ls-remote`
- 比较本地与远程 HEAD 或 tag
- 仅通知用户——绝不自动应用更新
