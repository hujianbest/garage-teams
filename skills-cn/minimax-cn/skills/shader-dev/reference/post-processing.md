# 后处理效果详细参考

该文件是对 [SKILL.md](SKILL.md) 的完整补充，涵盖先决条件、每个步骤的详细说明（内容和原因）、变体详细信息、深入的性能优化分析以及完整的组合建议。

## 先决条件

- GLSL 基础知识和 ShaderToy 环境（iResolution、iTime、iChannel、textureLod 等）
- 基本向量和矩阵运算
- 线性色彩空间和伽马校正之间的差异
- 纹理采样和UV坐标系
- 卷积的基本概念（核、权重、归一化）
- Multi-pass rendering concepts (Buffer A/B/C/D and Image pass in ShaderToy)

## 适用场景

当您完成场景的主要渲染并需要对结果进行屏幕空间图像增强时，请使用此技术。典型应用包括：

- **HDR 到 LDR 转换**：在场景中使用线性 HDR 照明后，需要色调映射将值压缩到可显示范围内
- **气氛增强**：晕影、颜色分级和胶片颗粒等效果可增强电影外观
- **发光和绽放**：模拟镜头绽放以在明亮区域周围产生柔和的光扩散
- **运动和散焦模糊**：通过运动模糊和景深模拟物理相机特性
- **抗锯齿**：后处理 AA 解决方案，例如 FXAA 和 TAA
- **色差和镜头效果**：色差和镜头眩光等光学模拟

## 核心原则

### 色调映射

将 HDR 线性颜色值从 [0, ∞) 映射到 LDR 显示范围 [0, 1]。核心数学模型：

- **Reinhard**：`color = color / (1.0 + color)`，简单的 S 曲线压缩
- **电影莱因哈德**：`q = (T²+1)·x² / (q + x + T²)`，具有白点 (W) 和肩部 (T) 参数
- **ACES**：行业标准，通过 3×3 矩阵将颜色转换到 ACES 颜色空间，然后应用有理多项式“(ax+b)/(cx+d)+e”进行非线性映射
- **一般有理多项式**：`(a·x²+b·x) / (c·x²+d·x+e)`，可以拟合各种色调曲线

### 高斯模糊

2D 高斯核`G(x,y) = exp(-(x²+y²)/(2σ²))`。由于可分离性，它可以分为两个一维通道（水平+垂直），将 O(n²) 减少到 O(2n)。

### 绽放

提取明亮像素（亮通阈值），然后应用多级高斯模糊并将结果添加回原始图像。多倍频程方法：逐步下采样+模糊，然后逐步合成，产生从窄到宽的光晕层。

### 小插图

根据像素到屏幕中心的距离减弱亮度。常用公式：
- **乘法**：`16·u·v·(1-u)·(1-v)`的幂
- **径向**：`1 - pow(dist * scale, exponent)`与强度混合

### 色差

模拟不同波长的镜头折射差异。对 R/G/B 通道使用不同比例因子对相同纹理进行采样，偏移量从中心到边缘逐渐增加。

## 实施步骤

### 第 1 步：色调映射 — 将 HDR 映射到可显示范围

**什么**：将渲染输出中的 HDR 线性颜色值压缩到 [0,1] 范围内。

**为什么**：物理上正确的照明计算产生的亮度值远远超过显示范围。直接夹紧会丢失高光细节。色调映射使用非线性曲线来保留阴影细节和高光过渡。

四种方法的比较：
- **莱因哈德**：最简单，适合初学者。单行“颜色/（1.0+颜色）”实现了S曲线压缩，但高光区域压缩得太厉害，缺乏​​平滑的“肩部”过渡。
- **Filmic Reinhard**：白点 (W) 参数控制最亮值的映射位置，肩部参数 (T2) 控制高光的压缩程度。 T2 值越高，高光过渡越柔和。
- **ACES**：行业标准方法。首先通过输入矩阵将线性 sRGB 转换为 ACES AP1 色彩空间，应用有理多项式非线性映射，然后通过输出矩阵转换回 sRGB。最准确的颜色表示，但计算成本稍高。
- **通用有理多项式**：具有 5 个可调参数的通用曲线，可以手动拟合任何色调曲线。最大的灵活性，但需要手动调整参数。

### 步骤 2：伽玛校正 — 线性空间到显示空间**内容**：将线性颜色值转换为 sRGB 伽玛空间，以便在显示器上正确显示。

**原因**：显示器亮度响应是非线性的（大约 γ=2.2）。直接输出线性值会显得太暗。伽玛校正通过“pow(1/2.2)”进行补偿。

注意事项：
- ACES 方法已经包含伽玛校正，因此不需要额外的步骤
- 某些管道使用 0.4545 (≈1/2.2) 作为伽玛值
- 色调映射后必须执行伽玛校正

### 步骤 3：对比度增强 — Hermite S 曲线

**内容**：对色调映射颜色应用 S 曲线以增强中间色调对比度。

**为什么**：色调映射后，图像可能会显得平坦。 S 形曲线使暗处更暗，亮处更亮，增加视觉冲击力。 “smoothstep”的三次 Hermite 基函数“3x² - 2x³”是一条自然的 S 曲线。

实施细节：
- 必须在伽玛校正后执行，当取值范围为[0,1]时
- 使用“clamp”确保输入在有效范围内
- `contrast_strength` 参数通过 `mix` 控制效果强度，0 表示无效果，1 表示完全效果
- `smoothstep(-0.025, 1.0, color)` 版本在黑暗中提供轻微的脚趾提升，避免纯黑色

### 步骤 4：颜色分级

**内容**：应用通道级调整来改变整体色调。

**为什么**：不同的色温和色调传达不同的情绪。暖色调（偏黄/橙）给人一种舒适的感觉，而冷色调（偏蓝/青）给人一种超然的感觉。

详细介绍四种方法：
- **每通道乘法**：最简单、最直接。 `vec3(1.11, 0.89, 0.79)` 增强红色通道，同时减少蓝色/绿色，产生暖色调。将系数交换为冷色调。
- **强力颜色分级**：通过更改每个通道的伽玛曲线来调整颜色。值 <1 使该通道变亮，>1 使该通道变暗。比乘法更温和，对中间调的影响更大。
- **HSV Hue Shift**：转换为HSV后，可以直接旋转色调并调整饱和度。适用于需要精确色调控制的场景。
- **去饱和混合**：将原始颜色与其亮度值（灰度）混合。较高的混合比例会产生更加褪色的外观，从而产生“电影”或“褪色”效果。

### 步骤 5：小插图

**什么**：使图像的边缘变暗，以引导观看者的焦点转向中心。

**为什么**：模拟真实镜头的光学渐晕，是一种经典的电影构图技术。

三种方法的比较：
- **方法 A（乘法，经典）**： `16·u·v·(1-u)·(1-v)` 在 UV 空间中构造一个抛物线曲面，其中心等于 1，角点等于 0。 power参数控制衰减速度，常用0.25。优点：计算量最少。缺点：固定矩形渐变形状。
- **方法 B（径向距离）**：基于像素到屏幕中心的欧几里得距离。考虑纵横比校正，产生椭圆形晕影。三个参数控制强度、起始半径和衰减陡度。
- **方法 C（逆二次衰减）**：`1/(1 + dot(p,p))` 产生非常自然的光学渐晕。平方两次会使衰减更加明显。 Smoothstep 混合控制效果强度。

### 步骤 6：高斯模糊 — 基本模糊

**什么**：对图像应用高斯卷积模糊。这是 Bloom 的基本构建块。

**为什么**：高斯内核是唯一既各向同性且可分离的平滑内核，可产生自然柔和的模糊。

实施细节：
- `normpdf` 计算高斯概率密度，其中 0.39894 ≈ 1/√(2π)
- KERNEL_SIZE 必须是奇数以确保中心对称
- 首先构建一个一维内核并利用对称性（`kernel[HALF+j] = kernel[HALF-j]`）
- Z 是归一化因子，确保所有权重之和为 1
- 2D 卷积通过两个嵌套循环实现，外积 `kernel[j] * kernel[i]` 构造 2D 权重
- 在生产中，使用可分离的方法（两个一维通道）以获得更好的性能

### 步骤 7：绽放 — HDR 发光

**内容**：从图像中提取明亮区域，应用多级模糊，然后将结果添加回来以创建辉光扩散效果。

**为什么**：人眼和相机镜头都会在强光源周围看到辉光。 Bloom 是增强图像“HDR 感觉”最有影响力的后处理效果。

实施细节：- 使用“textureLod”从 mipmap 的高 LOD 级别进行采样； GPU 硬件自动处理下采样模糊
- LOD 5/6/7 采样相当于大约 32x/64x/128x 下采样，产生从窄到宽的不同模糊半径
- 2x2 邻域超级采样（从 -1 到 1 循环）减少块效应
-“maxBloom”上限可防止极亮像素产生过度泛光
- `pow(bloom, vec3(1.5))` 应用伽玛调整以将绽放集中在明亮区域
- 注意：ShaderToy Buffers 默认不生成 mipmap；必须在频道设置中启用此功能

### 步骤 8：色差

**内容**：对具有不同 UV 比例的 R/G/B 通道进行采样以模拟镜头色散。

**为什么**：真正的镜头无法将所有波长的光聚焦到同一焦平面上。这种“不完美”实际上增加了图像的真实感和视觉趣味。

实施细节：
- 偏移方向是从屏幕中心计算的
- 在每次迭代中，R/G/B 通道均使用不同的比例因子进行采样
- 红色通道收缩（rf 减小），蓝色通道扩展（bf 增大），绿色通道几乎保持不变
- 收缩/膨胀率的差异产生分散效应，从中心到边缘逐渐增加
- 迭代实现以不同比例因子累积样本以模拟连续频谱
- CA_SAMPLES：更多样本产生更平滑的结果； 4-8通常就足够了

### 步骤 9：胶片颗粒

**什么**：叠加伪随机噪声来模拟胶片颗粒纹理。

**为什么**：微妙的随机噪声打破了数字图像的“完美”感觉，添加了有机纹理，并有助于减少色带。

两种实现方式：
- **哈希噪声**：一个简单的 `fract(sin(...) * 43758.5453)` 伪随机函数。乘以iTime以确保每帧噪声不同。大约 0.012 的强度看起来很自然。
- **拜耳矩阵有序抖动**：4x4 拜耳矩阵提供 17 级有序抖动。比随机噪声更均匀，特别适合消除 8 位色带。 `(dither - 0.5) * 4.0 / 255.0` 将抖动量限制为大约 ±2 个颜色级别。

### 第 10 步：运动模糊

**什么**：沿着每个像素的运动方向应用方向模糊。

**为什么**：静态框架缺乏运动感。运动模糊模拟快门曝光时物体运动的效果，使动画更流畅、更自然。

实施细节：
- 运动方向由速度缓冲区确定
- 沿运动方向均匀采样，权重线性递减（距离越远权重越低）
- MB_STRENGTH 控制模糊半径（在 UV 空间中）； 0.25 表示从像素到 25% 屏幕距离进行采样
- 32 个样本通常就足够了；随机抖动可以用更少的样本获得类似的结果

相机重投影方法：
- 需要深度缓冲区和前一帧的相机矩阵
- 将当前像素的世界坐标投影到前一帧的UV以获得运动矢量
-shutterAngle参数（0~1）控制模糊量
- 随机样本位置避免规则条纹伪影

### 第 11 步：景深

**什么**：根据像素深度和焦平面距离计算混乱圆（CoC），并使用散焦盘采样来模拟失焦模糊。

**为什么**：模拟真实的薄镜头模型，为焦平面外的物体产生柔和的散景，增强深度感知。

实施细节：
- **CoC模型**：基于薄透镜公式，CoC尺寸与像素深度偏离焦平面的程度成正比。光圈参数控制光圈大小，影响景深范围。
- **斐波那契螺旋采样**：黄金角（≈2.3998弧度）确保采样点均匀分布在磁盘上。 `sqrt(i)` 半径增量产生均匀的面积密度。
- **权重策略**：使用每个样本点自己的 CoC 作为权重，确保焦点清晰区域不会被失焦模糊“污染”。
- 64 个样本产生高质量的散景； 32 个足以满足大多数需要。

### 步骤 12：FXAA — 快速近似抗锯齿

**内容**：检测图像中的锯齿边缘并沿边缘应用方向模糊以消除锯齿。**为什么**：后处理AA不需要修改渲染管线，成本极低。 FXAA 通过亮度梯度检测边缘方向，并使用少量纹理样本进行方向模糊。

实施细节：
- 从 4 个对角邻居（西北、东北、西南、东南）和中心采样亮度
- 计算最终质量评估的亮度范围（lumaMin/lumaMax）
- 边缘方向根据水平/垂直亮度差异计算
- `dirReduce` 和 `rcpDirMin` 控制方向向量的缩放以防止过度模糊
- 两级采样策略：rgbA 在 1/3 和 2/3 位置采样，rgbB 在 -0.5 和 0.5 位置添加采样
- 最终决定：如果 rgbB 的亮度超出邻域范围（表示边缘交叉），则回退到 rgbA

## 变体详细信息

### 变体 1：多通道可分离布隆

与基本的单通道 mipmap 光晕不同：使用独立的 Buffer 进行可分离的高斯模糊（水平通道 + 垂直通道），提供更高的光晕质量和更好的控制。

**缓冲区 A 详细信息（水平模糊 + 下采样）**：
- `BLOOM_THRESHOLD`：亮度阈值；只有超过该值的像素才会进入布隆。值越低意味着参与的像素越多。
- `BLOOM_DOWNSAMPLE`：下采样因子； 3表示以1/3分辨率计算。减少计算量，同时扩大有效模糊半径。
- `BLUR_RADIUS`：模糊半径（以像素为单位）； 16 表示每个方向采样 16 个像素。
- 高斯权重“exp(-8.0 * d * d)”中的“-8.0”控制衰减速度；调整以改变模糊的“柔和度”。
- 边界检查 `xy.x >= int(iResolution.x) / BLOOM_DOWNSAMPLE` 确保仅在下采样区域内进行计算。

**缓冲区 B 细节（垂直模糊）**：
- 与缓冲区 A 的结构相同，只是采样方向从水平“ivec2(k, 0)”更改为垂直“ivec2(0, k)”
- 输入是缓冲区 A 的输出（iChannel0 绑定到缓冲区 A）
- 两个可分离模糊通道的组合相当于完整的 2D 高斯模糊

### 变体 2：ACES + 完整颜色管道

与基础版不同：采用完整的ACES RRT+ODT管线，包括色彩空间矩阵转换和内置sRGB gamma，适合追求影院级色彩的项目。

主要区别：
- 输入矩阵 m1 将线性 sRGB 转换为 ACES AP1 色彩空间
- 有理多项式 `(v*(v+a)-b) / (v*(c*v+d)+e)` 模拟 ACES RRT（参考渲染变换）
- 输出矩阵 m2 将 ACES AP1 转换回线性 sRGB
- 最终的“pow(..., 1/2.2)”执行 sRGB gamma 编码，因此使用此方法时不需要单独的 gamma 校正步骤

### 变体 3：物理自由度 + 运动模糊组合

与基本版本的不同之处：使用深度缓冲区和前一帧相机矩阵来实现物理上正确的景深+运动模糊，共享相同的采样循环。

按键设计：
- DoF 和运动模糊在同一个 for 循环中处理，避免了两次独立的采样过程
- `randomT` 哈希随机化每个样本点的时间位置，减少规则条纹伪影
- 运动模糊：通过“shutterAngle”在当前帧和前一帧 UV 之间进行插值
- DoF：斐波那契螺旋线偏移，偏移量由CoC控制
- 两种效果在堆叠后共享相同的“textureLod”样本，节省一半的带宽

### 变体 4：TAA 临时抗锯齿

与基本 FXAA 不同：利用多帧历史记录进行时域超级采样。每个帧都使用子像素抖动，与前一帧混合，并使用邻域颜色钳位来防止重影。

关键步骤说明：
1. **去抖动采样**：当前帧使用亚像素抖动进行渲染；采样期间，必须减去抖动偏移以恢复正确的 UV
2. **邻域限制**：3x3 邻域中颜色的最小/最大定义了“合理的颜色范围”。在此范围之外的历史帧颜色表示场景变化（遮挡/显示）
3. **重投影**：使用当前像素的世界坐标和上一帧的视图投影矩阵来计算上一帧中对应的UV位置
4. **混合策略**：当历史框架在合理范围内时，使用较高的权重（0.9）以实现时间稳定性；当超出范围时，使用0权重来充分利用当前帧并避免重影5. `blend = 0.9` 是可调整的：值越高越平滑，但更容易出现拖尾伪影

### 变体 5：镜头光晕 + 星爆

与基本版本的不同之处：在光晕之上叠加镜头光晕模拟，包括星爆和彩色鬼影。

关键技术讲解：
- **星爆图案**：`cos(angle * NUM_APERTURE_BLADES)` 在角度域中创建周期性图案，模拟光圈叶片的衍射。 `NUM_APERTURE_BLADES` 控制星爆点的数量。 “pow”控制星暴的锐度，离光源越远，星芒的清晰度就越不明显。
- **八角形鬼影**：沿着光轴（从太阳到屏幕中心的线）放置在反射位置的多个鬼影。 `smoothstep` 产生软边圆盘形状。
- **光谱颜色**：`wavelengthToRGB`将波长（nm）转换为RGB； `fract(ghostDist * 5.0)` 在鬼影内产生彩虹带，模拟真实镜头鬼影的色散效果。

## 深入的性能优化

### 1.可分离模糊代替2D卷积

一个11×11的2D高斯卷积需要121个样本；分成两个一维通道仅需要 22。这是所有模糊操作的主要优化。

数学基础：高斯核的可分性`G(x,y) = G(x) · G(y)`，意味着一个2D高斯核等于两个1D高斯核的外积。这意味着先执行水平模糊，然后执行垂直模糊（反之亦然）会产生与直接 2D 卷积相同的结果。

### 2.硬件Mipmap代替手动下采样

`textureLod(tex, uv, lod)` 利用 GPU 硬件的 mipmap 链进行免费下采样模糊，适合快速绽放。请注意，ShaderToy Buffers 默认情况下不会生成 mipmap（您需要在通道设置中启用“mipmap”）。

每个 mipmap 级别将分辨率减半，相当于 2x2 盒式过滤器。 LOD 5 对应于 32x 下采样，LOD 6 对应于 64x，LOD 7 对应于 128x。虽然盒式滤波器不是高斯核，但当多个级别组合时，结果接近高斯。

### 3. 模糊前下采样

Bloom 不需要以全分辨率计算。首先将图像下采样 2-4 倍，在低分辨率下模糊，然后双线性上采样回来。

优点：
- 计算量减少 4-16 倍（面积比）
- 相同的模糊内核大小以较低的分辨率覆盖更大的屏幕区域
- 上采样期间的双线性插值自动平滑结果

### 4. 减少样本数量

推荐样本数量：
- 运动模糊：16-32 个样本通常就足够了；使用随机抖动（时间抖动）而不是定期间隔来隐藏采样不足造成的条纹伪影
- DoF：32-64 斐波那契螺旋样本可产生高质量的散景。斐波那契螺旋比随机分布更均匀，避免聚类
- 色差：4-8 个样本可产生良好的结果，因为色差本质上是一种低频变化

### 5. 利用双线性插值实现自由模糊

两个纹素之间的采样会导致 GPU 硬件自动执行双线性混合，相当于 2 次平均。单个样本有效地从 4 个纹素获得加权信息。

应用：将 5-tap 高斯模糊优化为 3 个纹理样本（中心 1 个 + 每侧 1 个，侧面样本点放置在两个纹理像素之间）。

### 6.条件编译

使用“#define”开关来控制每个后处理模块。禁用不需要的效果的成本为零——预处理器完全删除代码，不生成任何指令。```glsl
#define ENABLE_BLOOM 0  // Disable bloom; the branch code is completely absent after compilation
```### 7. 避免分支

后处理中的 if/else 语句应尽可能转换为数学形式，如“mix”/“step”/“smoothstep”，避免 GPU 扭曲发散。

例子：```glsl
// Bad: if/else branching
if (brightness > threshold) color = bright_path; else color = dark_path;

// Good: mathematical form
float t = step(threshold, brightness);
color = mix(dark_path, bright_path, t);
```## 组合建议

### 1. Bloom + Tone Mapping（最基本的组合）

Bloom 在线性 HDR 空间中计算，添加到场景中，然后应用色调映射。 **顺序不能颠倒**——在LDR空间中进行Bloom意味着高光已经被钳制，并且Bloom无法正确提取超亮像素。```glsl
// Correct order
color += bloom;          // Add bloom in HDR space
color = tonemap(color);  // Then tone map
color = pow(color, vec3(1.0/2.2)); // Finally gamma
```### 2. TAA + 运动模糊 + DoF（物理相机模拟）

TAA 首先消除锯齿，然后 DoF 和运动模糊可以共享采样循环。 TAA 的子像素抖动还可以补充运动模糊的时间抖动。

建议的管道顺序：
1. TAA（Buffer D）：混合当前帧+历史帧
2. DoF + Motion Blur（图像通道）：共享采样循环
3.其他后续影响

### 3.色差+暗角+胶片颗粒（镜头模拟三重奏）

这三种效果都模拟了物理镜片缺陷；组合后，图像具有强烈的“真实镜头”感觉。

执行顺序：
1. 色差（CA）在采样阶段完成——直接替换正常的`texture()`调用
2. 在所有颜色处理后乘法应用晕影
3. 最后添加谷物

### 4.颜色分级+色调映射+对比度（颜色管道）

颜色分级（乘法/功率调整）在线性空间中完成，色调映射处理 HDR 压缩，S 曲线对比度在伽玛空间中应用。这三个步骤的顺序决定了最终的色彩风格。

要点：线性空间中的颜色分级会产生最自然的结果，因为感知的亮度关系在线性空间中是正确的。

### 5. Bloom + 镜头光晕（电影灯光效果）

Bloom 提供柔和的高光扩散；镜头光晕提供星爆和鬼影。两者共享相同的亮通提取结果，但耀斑计算方向图案，而光晕是各向同性模糊。

### 6. 多通道完整管道（生产级）

推荐的生产级管道：
- **缓冲区 A**：场景渲染 + 速度/深度编码（将运动矢量和深度打包到 Alpha 通道或附加纹理中）
- **缓冲区 B**：布隆下采样 + 水平模糊（缓冲区 A 的亮通输出上的水平高斯）
- **缓冲区 C**：布隆垂直模糊（缓冲区 B 上的垂直高斯，完成可分离布隆）
- **Buffer D**：TAA（当前帧+历史帧混合，需要读取Buffer D自身的历史输出）
- **图像**：最终合成 - DoF + 运动模糊 + Bloom 合成 + 色调映射 + 颜色分级 + 晕影 + 颗粒 + 抖动
