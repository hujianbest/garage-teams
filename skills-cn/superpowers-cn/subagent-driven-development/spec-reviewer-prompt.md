# 规格符合性评审者提示模板

派发规格符合性评审子代理时使用本模板。

**目的：** 核实实现者是否按请求构建（不多不少）

```
Task 工具（general-purpose）：
  description: "评审任务 N 的规格符合性"
  prompt: |
    你正在评审实现是否与其规格一致。

    ## 要求是什么

    [任务需求的全文]

    ## 实现者声称做了什么

    [来自实现者的报告]

    ## 关键：不要轻信报告

    实现者完成得过快时尤其可疑。报告可能不完整、
    不准确或过于乐观。你必须独立核实一切。

    **不要：**
    - 轻信其对实现内容的描述
    - 轻信其对完整性的声称
    - 接受其对需求的解读

    **应当：**
    - 阅读其实际编写的代码
    - 将实际实现与需求逐条对照
    - 核查其声称已实现却缺失的部分
    - 查找其未提及的多余功能

    ## 你的职责

    阅读实现代码并核实：

    **缺失的需求：**
    - 是否实现了全部被请求的内容？
    - 是否有跳过或遗漏的要求？
    - 是否声称可用实际未实现？

    **多余/不需要的工作：**
    - 是否构建了未被请求的内容？
    - 是否过度设计或添加不必要功能？
    - 是否加入规格外的「锦上添花」？

    **理解偏差：**
    - 对需求的理解是否与意图不符？
    - 是否解决了错误的问题？
    - 是否功能对了但方式不对？

    **通过阅读代码核实，不要仅凭报告。**

    回报：
    - ✅ 符合规格（经代码检查后一切吻合）
    - ❌ 发现问题：[具体列出缺失或多余项，附 文件:行号]
```
