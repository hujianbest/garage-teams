# 执行计划

## 概述

加载计划、批判性审阅、执行全部任务，完成后汇报。

**开场声明：** 「我正在使用 executing-plans 技能来实现本计划。」

**说明：** 请告诉你的真人协作者：Superpowers 在能使用子代理时效果好得多。若在支持子代理的平台（如 Claude Code 或 Codex）上运行，工作质量会明显更高。若子代理可用，请使用 superpowers:subagent-driven-development，而不要使用本技能。

## 流程

### 第 1 步：加载并审阅计划
1. 阅读计划文件
2. 批判性审阅 — 标出对计划的疑问或顾虑
3. 若有顾虑：在开始前提请真人协作者讨论
4. 若无顾虑：创建 TodoWrite 并继续

### 第 2 步：执行任务

对每个任务：
1. 标为 in_progress
2. 严格按计划中的每一步执行（计划已拆成小步）
3. 按计划运行指定的验证
4. 标为 completed

### 第 3 步：完成开发

所有任务完成且验证通过后：
- 声明：「我正在使用 finishing-a-development-branch 技能来完成这项工作。」
- **必选子技能：** 使用 superpowers:finishing-a-development-branch
- 按该技能验证测试、呈现选项并执行所选路径

## 何时停止并求助

**遇到下列情况立即停止执行：**
- 遇到阻塞（缺少依赖、测试失败、指令不清）
- 计划存在无法开工的严重缺口
- 不理解某条指令
- 验证反复失败

**宁可澄清也不要猜测。**

## 何时回到更早步骤

**回到审阅（第 1 步）当：**
- 协作者根据你的反馈更新了计划
- 基本思路需要重新考虑

**不要硬闯阻塞** — 停下来提问。

## 请记住
- 先批判性审阅计划
- 严格按计划步骤执行
- 不要跳过验证
- 计划要求时引用相应技能
- 受阻时停下，不要猜
- 未经用户明确同意，永远不要在 main/master 上开始实现

## 集成

**所需工作流技能：**
- **superpowers:using-git-worktrees** — 必选：开始前建立隔离工作区
- **superpowers:writing-plans** — 生成本技能所执行的计划
- **superpowers:finishing-a-development-branch** — 全部任务完成后收尾开发
