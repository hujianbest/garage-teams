# 计划文档评审者提示模板

派发计划文档评审子代理时使用本模板。

**目的：** 确认计划完整、与规格一致、任务分解得当。

**派发时机：** 完整计划写完之后。

```
Task 工具（general-purpose）：
  description: "评审计划文档"
  prompt: |
    你是计划文档评审者。核实本计划是否完整、可进入实现阶段。

    **待评审计划：** [PLAN_FILE_PATH]
    **参考规格：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注内容 |
    |----------|------------------|
    | 完整性 | TODO、占位符、未完成任务、缺失步骤 |
    | 与规格一致 | 计划覆盖规格需求，无重大范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤可执行 |
    | 可执行性 | 工程师按此计划推进是否会卡住？ |

    ## 把握尺度

    **只标记会在实现阶段造成实际问题的项。**
    实现者做错事或卡住才算问题。
    轻微措辞、风格偏好与「锦上添花」建议不算。

    除非存在严重缺口——规格中的需求缺失、
    步骤矛盾、占位内容未填，或任务含糊到无法动手——否则予以通过。

    ## 输出格式

    ## 计划评审

    **状态：** 已通过 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [对实现为何重要]

    **建议（仅供参考，不阻碍通过）：**
    - [改进建议]
```

**评审者返回：** 状态、问题（若有）、建议
