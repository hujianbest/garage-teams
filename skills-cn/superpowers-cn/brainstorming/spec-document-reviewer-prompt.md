# 规格文档评审提示模板

派发规格文档评审子代理时使用本模板。

**目的：** 确认规格完整、一致，且已可进入实现计划阶段。

**派发时机：** 规格文档已写入 docs/superpowers/specs/

```
Task 工具（general-purpose）：
  description: "评审规格文档"
  prompt: |
    你是规格文档评审者。核实本规格是否完整、可进入计划阶段。

    **待评审规格：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注内容 |
    |----------|------------------|
    | 完整性 | TODO、占位符、「待定」、未完成章节 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 需求是否含糊到会让人做错实现 |
    | 范围 | 是否聚焦到单一计划可覆盖——而非多个独立子系统 |
    | YAGNI | 未请求的功能、过度设计 |

    ## 把握尺度

    **只标记会在制定实现计划时造成实际问题的项。**
    缺失章节、矛盾，或含糊到可作两种解读的需求——这些算问题。
    轻微措辞、风格偏好与「某节不如其他节细」不算。

    除非存在会使计划不可靠或产生严重缺陷的缺口，否则予以通过。

    ## 输出格式

    ## 规格评审

    **状态：** 已通过 | 发现问题

    **问题（如有）：**
    - [章节 X]：[具体问题] - [对计划阶段为何重要]

    **建议（仅供参考，不阻碍通过）：**
    - [改进建议]
```

**评审方返回：** 状态、问题（若有）、建议
