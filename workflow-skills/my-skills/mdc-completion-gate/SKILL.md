# MDC 完成门禁

在宣告完成之前，必须先确认有足够证据。

## 铁律

没有针对当前最新代码状态的验证证据，就不能宣称任务完成。

如果本轮流程里没有亲自运行验证命令，就不能诚实地宣称完成。

## 在以下动作前必须使用本 Skill

- 说任务已经完成
- 更新进度或状态为已完成
- 切换到下一个任务
- 编写带有“已完成”含义的提交或交付说明

## 工作流

### 1. 明确你要宣告的结论

明确写出你准备宣告什么，例如：

- 测试通过
- 功能行为正常
- 缺陷已修复
- 任务已完成

### 2. 确定证明该结论的命令

选择真正能证明该结论的命令。

不要用更弱的证据替代。

### 3. 执行最新验证

立即运行完整验证命令。

### 4. 阅读完整结果

检查：

- 退出码
- 失败数量
- 输出是否真的支持该结论

### 5. 对结论做门禁判断

- 如果证据支持结论，则允许完成
- 如果不支持，则说明真实状态并回退到 `mdc-implement`

## 输出格式

请严格使用以下结构：

```markdown
## 结论

PASS | REVISE | BLOCKED

## 已验证结论

- 结论项

## 证据

- 命令与结果摘要

## 下一步

`done-for-current-task` | `mdc-implement`
```

## 判定规则

只有当准备宣告的“完成”结论被最新证据直接支持时，才返回 `PASS`。

如果证据不足以支持该结论，或者仍需继续实现，则返回 `REVISE`。

如果由于环境或工具链问题无法运行正确的验证命令，则返回 `BLOCKED`。

## 反模式

- 说“现在应该算完成了”
- 依赖旧输出
- 把主观感觉当成证据
- 认为评审通过就等于运行成功
- 因为做烦了就直接进入下一步

## 完成条件

只有在用最新证据批准完成结论，或明确拒绝该结论并把工作退回修订后，这个 skill 才算完成。
