# MDC 代码评审

评审当前任务的实现代码。

## 目的

这个 skill 用于在测试质量完成评审后，判断实现本身是否可靠。

它不能替代需求评审或设计评审。

## 评审清单

### 1. 正确性

- 代码是否看起来满足当前任务目标？
- 是否明显漏掉了边界情况？

### 2. 设计一致性

- 实现是否仍然符合已批准设计？
- 当前任务是否引入了意外的架构漂移？

### 3. 可读性与可维护性

- 命名是否清晰易懂？
- 逻辑是否存在不必要的复杂度？
- 是否有单处承担过多职责的情况？

### 4. 错误处理

- 常见失败路径是否处理得当？
- 实现是否引入了脆弱假设？

## 输出格式

请严格使用以下结构：

```markdown
## 结论

PASS | REVISE

## 发现项

- [severity] finding

## 代码风险

- item

## 下一步

`mdc-traceability-review` | `mdc-implement`
```

## 判定规则

只有当当前任务实现足够合理、可以进入下一步验证时，才返回 `PASS`。

如果在正确性、可维护性或设计一致性方面仍有问题需要先修正，再返回 `REVISE`。

## 反模式

- 在这里重复需求评审意见
- 只因为测试是绿的就直接通过代码
- 因为任务小就忽略设计漂移

## 完成条件

只有在给出明确结论和唯一下一步之后，这个 skill 才算完成。
