# MDC 测试评审

评审当前任务的测试。

## 目的

这个 skill 用于判断测试是否真的在验证行为，而不只是给任务表面上加了一层浅覆盖。

## 评审清单

### 1. 失败优先纪律

- 在实现前，是否先出现了失败测试？
- 这个失败是否准确代表了缺失行为？

### 2. 行为价值

- 测试验证的是行为，而不是实现细节吗？
- 断言是否有意义？

### 3. 覆盖形态

- 在适用场景下，是否至少覆盖了基本异常路径或边界路径？
- 重要任务行为是否都被体现出来？

### 4. 测试质量风险

- 过度使用 mock
- 断言过弱
- 命名模糊
- 测试与实现细节耦合过紧

## 输出格式

请严格使用以下结构：

```markdown
## Verdict

PASS | REVISE

## Findings

- [severity] finding

## Test Quality Gaps

- item

## Next Step

`mdc-code-review` | `mdc-implement`
```

## 判定规则

只有当这些测试对当前任务真正有验证价值时，才返回 `PASS`。

如果测试质量太弱，尚不足以支撑对任务结果的信任，则返回 `REVISE`。

## 反模式

- 因为“有测试”就直接通过
- 忽略没有失败优先信号的问题
- 把浅层断言当成足够证据

## 完成条件

只有在给出明确结论和唯一下一步之后，这个 skill 才算完成。
