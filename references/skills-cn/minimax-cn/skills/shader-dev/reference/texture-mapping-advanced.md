# 高级纹理映射详细参考

## 先决条件
- 屏幕空间导数（`dFdx`、`dFdy`）
- `textureGrad()`函数使用
- 基本光线行进

## 三平面与双平面成本分析

|方面|三平面|双平面|
|--------|------------|----------|
|纹理获取 | 3 | 2 |
| ALU 运算 |降低|更高（轴选择）|
|带宽|更高 |降低|
|视觉质量|基线|当量（k≥8）|
|最适合 |带宽丰富的 GPU |移动，带宽有限 |

现代 GPU 通常受到带宽限制，而不是 ALU 限制，这使得双平面成为更好的默认选择。

### 权重重新映射数学

双平面权重公式“clamp((w - 0.5773) / (1.0 - 0.5773), 0, 1)”确保：
- 在与一个轴对齐的法线处：权重 = 1.0（干净投影）
- 2 个轴相等的 45° 对角线处：平滑过渡
- 在立方体对角线处 (1/√3 ≈ 0.5773)：权重 = 0.0，但这是需要第三个（废弃的）投影的点 - 双平面的近似误差在这里最大，但视觉上可以接受

### 梯度传播

使用“textureGrad()”而不是“texture()”至关重要，因为：
1. 轴选择（`ma`、`me`）在投影边界处创建 UV 不连续性
2. 硬件“texture()”根据隐式导数计算 mip，该导数在不连续处→可见接缝处出现峰值
3. 带有手动传播的“dFdx(p)”、“dFdy(p)”的“textureGrad()”绕过了这一点，保持渐变在边界上平滑

## 射线微分数学

### 问题陈述
在光栅化中，纹理坐标的“dFdx”/“dFdy”自然起作用，因为相邻像素映射到附近的表面点。在光线行进中，相邻像素可能会击中完全不同的对象→损坏的 mip 选择。

### 解决方案：切平面相交

鉴于：
- 主光线以正常的“nor”在“pos”处撞击表面
- 相邻像素光线“rd_neighbor”源自“ro_neighbor”

相邻光线与切平面在“pos”处的交点：```
t_neighbor = dot(pos - ro_neighbor, nor) / dot(rd_neighbor, nor)
pos_neighbor = ro_neighbor + rd_neighbor * t_neighbor
```差异“pos_neighbor - pos”给出了命中点处一个像素的世界空间足迹。

### 对于透视相机（常见情况）```
ro is the same for all pixels, only rd varies:
dposdx = t * (rdx * dot(rd, nor) / dot(rdx, nor) - rd)
dposdy = t * (rdy * dot(rd, nor) / dot(rdy, nor) - rd)
```其中“rdx = rd + dFdx(rd)”和“rdy = rd + dFdy(rd)”。

### 纹理坐标的链式法则
如果纹理映射函数为“uv = f(pos)”：```
duvdx = Jacobian(f) × dposdx
duvdy = Jacobian(f) × dposdy
```对于简单的平面映射 `uv = pos.xz`：```
duvdx = dposdx.xz
duvdy = dposdy.xz
```## 纹理重复理论

### 为什么平铺可见
人类视觉擅长检测：
1. **周期性图案**：规则的网格对齐
2. **独特的特征**：重复相同的独特斑点/标记
3. **相位对齐**：所有图块以相同相位开始

### 打破重复
每种方法针对不同的线索：
- **随机偏移**（方法 A）：破坏相位对齐，4 次读取
- **Voronoi 混合**：完全打破网格结构，9 次获取（昂贵）
- **虚拟模式**（方法B）：廉价地破坏独特的功能，2次获取

方法 B 更适合实时使用 — 低频索引变化是缓存友好的，并且两个纹理读取共享局部性。
