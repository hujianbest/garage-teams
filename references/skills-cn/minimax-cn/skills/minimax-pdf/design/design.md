# 设计系统

审美层。在接触任何脚本之前请阅读本文。
该文件回答了“它应该是什么样子以及为什么”。

---

## 唯一规则

每个设计决策都必须**植根于文档的内容和目的**。
深青色+奶油色并不“专业”。衬线+米色并不“优雅”。
因适合内容而选择的颜色总是优于所选择的颜色
因为它看起来很安全。

### 重音选择规则

- **仅一种强调色。** 使用两种强调色可分散视觉能量。
- 重音出现在：覆盖几何元素、剖面规则、标注左边框、
  表头背景，页眉规则。无处可去。
- 强调与封面背景的对比度必须至少为 4.5:1 (WCAG AA)。
- 不要默认为蓝色。蓝色是人工智能生成的文档中最滥用的重音。

### 颜色配对反模式（切勿使用这些）

| ❌避免 |为什么 |
|---|---|
|白底紫色渐变|默认的人工智能美学——立即发出“生成”的信号 |
|海军蓝+金色|过度使用的企业陈词滥调|
|全黑背景|打印效果很差，感觉很有攻击性|
|系统颜色超过3种 |视觉噪音|
|正文上的重音 |破坏可读性 |

## 排版系统

### 字体配对逻辑

最多两种字体。总是。

|角色 |标准|不错的选择（系统安全）|
|---|---|---|
|显示（封面标题，H1）|特色鲜明、对比强烈、分量重| Times New Roman，佐治亚州（衬线体）|
|文本（正文、标题、UI）| 10–11pt 可读性强 | Helvetica、Arial（无字体）|

封面字体通过封面 HTML 中的“@import url(...)”实时加载 — Playwright
在渲染时获取它们，没有本地缓存。正文页面始终使用系统字体
（Times-Bold / Helvetica）通过 ReportLab — 一致且离线安全。

按心情配对（仅涵盖 HTML - 正文始终使用系统字体）：
- 权威：“Playfair Display”/“IBM Plex Sans”
- 自信：“Syne”/“Nunito Sans”
- 富有表现力：“弗朗西斯”/“国际米兰”
- 学术：“EB Garamond”/“Source Sans 3”
- 干净：“DM Serif Display”/“DM Sans”
- 克制：“鸬鹚加拉蒙”/“约斯特”
- 粗体：“Barlow Condensed”/“Barlow”
- 动态：“蒙特塞拉特”/“蒙特塞拉特”
- 古典：“鸬鹚”/“Crimson Pro”
- 社论：“Bebas Neue”/“Libre Franklin”
- 正文后备（始终）：“Times-Bold”/“Helvetica”（ReportLab 系统字体）

### 类型比例

所有尺寸均以磅为单位。 `palette.py` 使用此比例来填充 `tokens.json`。

|代币|尺寸|领先|用途 |
|---|---|---|---|
| `显示` | 54点| 1.0 |封面标题|
| `h1` | 22点| 1.3 | 1.3章节标题 |
| `h2` | 15分| 1.4 | 1.4小节标题 |
| `h3` | 11.5 分 | 1.5 | 1.5子小节 |
| `身体` | 10.5 分 | 1.6 | 1.6主要散文 |
| `标题` | 8.5 分 | 1.4 | 1.4图/表标题|
| `元` | 8分| 1.3 | 1.3页眉/页脚文本 |

### 间距系统

页边距和节奏是“设计外观”与“印刷外观”的区别。

|代币|价值|笔记|
|---|---|---|
| `margin_outer` | 2.8 厘米 |左/右页边距 |
| `margin_top` | 2.8 厘米 |顶页边距 |
| `margin_bottom` | 2.5厘米|底页边距 || `section_gap` | 26点| H1 之前的空格 |
| `para_gap` | 8分|段后空格|
| `线间隙` | 17点|正文前导 |

切勿使用 ReportLab 的默认边距（太紧）。始终明确设置。

## 封面设计

封面是最重要的一页。它决定读者是否信任该文档。

### 十三种封面图案

`cover.py` 根据 `tokens.json["cover_pattern"]` 选择一个。

**1. `fullbleed`** — 用于：`report`、`general`
- 深背景填充页面的 100%
- 标题：大号，左对齐，占页面上部 60%
- 强调：细水平尺+右上角条
- 点网格背景纹理（微妙，8–10% 不透明度）
- 页脚带：作者 + 日期元数据
- 字体：Playfair Display / IBM Plex Sans

**2. `split`** — 用于：`提案`
- 左侧 42% 面板：纯封面颜色、标题 + 作者
- 右58%：灰白色，点状网格装饰
- 强调色的硬垂直分割线
- 无渐变——纯平面几何
- 字体：Syne / Nunito Sans

**3. `typography`** — 用于：`resume`、`academic`
- 白色/灰白色背景
- 名称或标题为超大显示类型 (60–80pt)，左对齐
- 第一个单词为强调色，其余为深色
- 标题栏下方的细线
- 字体：DM Serif Display / DM Sans（简历）·EB Garamond / Source Sans 3（学术）

**4. `atmospheric`** — 用于：`投资组合`
- 近乎黑色的背景
- 柔和的径向发光强调色（右上象限）
- 标题居左居中，最多 2 行
- 标题下方重音的简短规则
- 低不透明度的点网格纹理
- 字体：Frances / Inter

**5. `minimal`** — 用于：`minimal`
- 近乎白色的背景，8px的左强调栏是唯一的颜色
- 非常大、轻量级显示类型的标题（300 重量）
- 细线规则，作者+日期作为单条静音线
- 没有别的了——酒吧承担了所有的视觉工作
- 字体：Cormorant Garamond / Jost

**6。 `stripe`** — 用于：`stripe`
- 页面分为三个水平带：重音/深色/浅色
- 顶带：类别标签；中：白色超大标题；底部：元数据
- 硬边、无渐变、无纹理 — 报纸/品牌海报美学
- 字体：Barlow Condensed / Barlow

**7. `对角线`** — 用于：`对角线`
- SVG 多边形对角剪切页面：左上深色，右下浅色
- 强调色边缘线描绘对角线切割
- 暗区标题，亮区元数据
- 字体：蒙特塞拉特/蒙特塞拉特

**8. `frame`** — 用于：`frame`
- 白色/奶油色背景，带有嵌入矩形边框（距边缘 1.2 像素、28 像素）
- 框架顶部和底部内部的强调条；小口音角方块
- 标题在框架空间居中、居中对齐、经典权重
- 正式、永恒——年度报告、法律文件、学术论文
- 字体：Cormorant / Crimson Pro

**9. `编辑`** — 用于：`编辑`
- 标题的幽灵首字母以 5% 不透明度填充右上角 — 视觉纹理
- 5px 重音顶栏；压缩格式的全角大写标题
- 标题全部大写，非常大（80px），左齐平
- 页脚规则+作者/日期元数据
- 字体：Bebas Neue / Libre Franklin

**10. `杂志`** — 用于：`杂志`
- 温暖的奶油色/亚麻背景；完全居中、垂直堆栈布局
- 组织/公司名称采用小间距大写字母+下方 2 像素重音规则（顶部锚点）
- 大粗体衬线标题（52px）居中；标题下的短重音规则
- 斜体字幕；可选的“cover_image” URL 呈现为居中的英雄缩略图
- 可选的“摘要”字段：带有粗体“摘要：”标签的对齐文本块
- 以强调色显示的作者姓名（大号、粗体）；下面的日期
- 字体：Playfair Display / EB Garamond

**11. `darkroom`** — 用于：`darkroom`
- 与“杂志”相同的居中堆栈布局，但深海军背景，白色文本
- 组织名称+半透明白色规则；重音规则去饱和
- 英雄图像（如果提供）获得“灰度（20％）亮度（0.9）”过滤器
- 字体：Playfair Display / EB Garamond

**12. `terminal`** — 用于：`terminal`
- 近乎黑色的背景；霓虹绿口音；空间 整个单色等宽
- 网格覆盖：以 48 像素间隔的微弱水平线 + 垂直线（7% 不透明度）
- 左上角状态标签：绿点 + `SYSTEM_REPORT // <日期>`
- 括号内的标题（左边框 + 上边框 + 伪元素角）
- 字幕以“>”为强调色前缀
- 留下抽象文字；作者封锁权利；底部状态栏 (UTF-8 / Ln 1)
- 字体：Space Mono / Space Mono

**13。 `poster`** — 用于：`poster`- 白色背景；厚 52px 的左侧边栏带有强调色（通常接近黑色）
- 标题：96 像素、900 字重、全部大写、压缩 — 主导视觉元素
- 标题下方采用打字机字体的副标题；细 2px 规则作为分隔符
- 规则下 Courier Prime 等宽字体中的作者 + 元
- 可选的`cover_image`渲染为260×340灰度缩略图，右对齐
- 带有白色水平线的强调方形图标块（右下）
- 字体：Barlow Condensed / Courier Prime

### 可选标记：`cover_image`

模式“magazine”、“darkroom”和“poster”接受可选的“cover_image”
包含图像的绝对 URL 或“file://”路径的令牌。
图像通过 `<img src="...">` 渲染 — Playwright 在渲染时获取它。
如果省略，则简单地跳过图像区域（布局优雅地调整）。

### 涵盖 CSS 要求（对于 Playwright 渲染至关重要）

这三个规则必须出现在每个封面 HTML 文件中，否则输出将具有
白色边框/尺寸不正确：```css
body { margin: 0; padding: 0; }
html, body { width: 794px; height: 1123px; overflow: hidden; }
```不需要“@page”规则 - Playwright 通过“pdf()”调用处理页面大小。
不要使用 CSS `background-image` 作为纹理 - 使用内联 SVG 或 `<canvas>`。
对于分层元素，始终使用“position:absolute”+“z-index”。

### 什么总是杀死封面

- 标题在白色背景上居中，下方有一条细水平线
- 从一种颜色到另一种颜色的渐变（读作 PowerPoint，而不是印刷设计）
- 文本上的阴影
- 不止一种强调色
- 表情符号或图标字体（在无头 Chromium 上无提示地失败）

## 内页规则

### “约束”在实践中意味着什么

每个设计决策都应该删除一些东西，而不是添加一些东西。
当没有任何内容需要删除时，页面就完成了。

- 强调色仅出现在章节规则上 - 不在标题上，不在项目符号上
- 无卡片组件（带彩色标题的边框框）
- 除标注框（最大 4 像素）外，任何东西都没有圆角
- 任何地方都没有阴影
- 表格：标题行强调，交替行色调，除外框外没有网格线
- 标注框：左边框为重音（4px），非常浅的色调背景，无图标

### 页眉/页脚

标题：文档标题（左，7.5pt，静音）+重音规则（1.5pt，下方全角）
页脚：作者姓名（左，7.5pt，静音）+页码（右，7.5pt，静音）+上面的浅色规则

## 质量栏

如果设计师不会不好意思将 PDF 交给客户，那么 PDF 就可以通过。
具体来说：

- 封面具有清晰的视觉识别，不是“通用人工智能输出”
- 正文文本无需眯眼即可在一臂距离处阅读
- 每个页面看起来都属于同一个文档
- 没有元素从边缘溢出或与另一个元素重叠
- 页码存在且正确
- 强调色平均每页出现次数少于 8 次

### 数学公式指导

**输入语法：** 标准 LaTeX 数学符号 — `\frac{}{}`、`\int`、`\sum`、`\alpha`、`^`、`_` 等。
**渲染引擎：** matplotlib mathtext — 纯Python，无需LaTeX编译器，无需浏览器。

|语法示例 |呈现为 |
|---|---|
| `E = mc^2` |内联表达式|
| `\frac{\sqrt{\pi}}{2}` |分数|
| `\int_0^\infty e^{-x^2} dx` |积分 |
| `\sum_{i=1}^{n} x_i` |总结|
| `\alpha + \beta = \gamma` |希腊字母|**限制：** matplotlib mathtext 涵盖最常见的表达式，但不涵盖高级 LaTeX 环境（“align”、“cases”、“matrix”）。将复杂的多行证明拆分为多个“数学”块。

**后备：** 如果未安装 matplotlib，则以代码样式呈现为“表达式”。运行“make.sh fix”进行安装。

**方程标签：** `"label": "(1)"` — 在公式旁边右对齐呈现。

### 图表指导

**完全用 Python 渲染** — 不需要外部图表服务、图像文件或互联网。

|图表类型 |使用案例 |必填字段 |
|---|---|---|
| `酒吧` |比较离散类别 | `标签`、`数据集` |
| `线` |随时间变化的趋势或有序类别 | `标签`、`数据集` |
| `馅饼` |部分到整体的构图| `标签`、`数据集[0].值` |

- 颜色源自文档强调以确保视觉一致性 - 不要设置自定义颜色。
- 多系列：将多个对象添加到“数据集”，每个对象都有一个“标签”和“值”数组。
- 图自动编号：设置`"figure": true`（默认）或`"figure": false`来抑制。

### 流程图指导

**节点形状：**

|形状|用于|
|---|---|
| `直`（默认）|工艺步骤|
| '钻石' |决定/条件|
| “椭圆形”或“终端” |开始/结束 |
| `平行四边形` |输入/输出|

- 节点按输入顺序放置（从上到下）。这控制了布局。
- 前边缘绘制直箭头；后边缘（到较早的节点）绘制弧形。
- 保持标签简短（最多 3-5 个字）——图表为 A4 列宽，比例为 78%。
- 图形自动编号与图表相同。

### 参考书目指导

- `id` 字段是参考标签 — 使用数字（“1”、“2”）或字母数字（“Smith23”）。
- 文本应采用一致的引用样式（APA、芝加哥等）——渲染器不强制样式。
- “标题”字段默认为“参考文献”。设置 `"title": ""` 来隐藏标题。
- “参考书目”块始终以新的章节标题+重音规则开头。

### 图片/图形指导

- 首选格式：PNG、JPEG
- 如果比文本列宽则缩小；从未扩大规模
- `figure` 阻止自动编号； `image` 块不使用 `figure` 来表示编号的图形
- 如果渲染时文件不存在，则会替换“[图像未找到]”占位符

### 代码块指导

- 准确保留空格 — 不要缩进 JSON 值中的代码
- 可选的“语言”字段在块上方呈现一个小语言标签（例如，““语言”：“python””）
- 无语法高亮（设计使然）——符合约束原则
- A4 列宽的行数控制在约 90 个字符以内
