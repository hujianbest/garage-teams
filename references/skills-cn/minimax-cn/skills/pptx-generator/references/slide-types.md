# 幻灯片页面类型

将**每张幻灯片**分类为**这 5 种类型中的**一种**：

## 1. 封面页

- **用途**：开场+音调设置
- **内容**：大标题、副标题/演示者、日期/场合、强烈的背景/主题

### 布局选项

**左右不对称布局**
- 文本集中在一侧，图像集中在另一侧
- 最适合：企业演示、产品发布会、专业报告```
|  Title & Subtitle  |    Visual/Image    |
|  Description       |                    |
```**中心对齐布局**
- 内容以背景图像为中心
- 最适合：励志演讲、活动演示、创意演讲```
|                                        |
|           [Background Image]           |
|              MAIN TITLE                |
|              Subtitle                  |
|                                        |
```### 字体大小层次结构

|元素|推荐尺寸 |与基础的比率|
|--------|-----------------|------------|
|主标题 | 72-120 像素 | 3x-5x |
|字幕| 28-40 像素 | 1.5x-2x|
|支持文字| 18-24 像素 | 1x（基础）|
|元信息（日期、姓名）| 14-18 像素 | 0.7x-1x |

**关键原则：**
1. **戏剧性对比**：主标题应至少比副标题大2-3倍
2. **视觉锚点**：最大的文本成为焦点
3. **可读的层次结构**：观众应该立即明白什么是最重要的
4. **避免相似**：切勿让相邻文本元素的大小相差在 20% 以内

### 内容元素

1. **主标题** — 始终必需，最大字体
2. **副标题** - 当需要额外的上下文时（明显小于标题）
3. **图标**——当它们强化主题时
4. **日期/事件信息** — 相关时（最小文本）
5. **公司/品牌徽标** — 代表一个组织时
6. **演讲者姓名** — 用于主题演讲（小而微妙）

### 设计决策

考虑：目的（企业/教育/创意）、受众、基调、内容量、所需的视觉资产。

### 工作流程

1. **分析**：了解主题、受众、目的
2. **选择布局**：根据内容选择
3. **编写幻灯片**：使用 PptxGenJS。使用形状和 SVG 元素来增加视觉趣味。
4. **验证**：生成预览为 `slide-XX-preview.pptx`。使用“python -m markitdown slip-XX-preview.pptx”提取文本，验证所有内容，并且不保留占位符文本。

## 2. 目录

- **用于**：导航 + 期望设置（3-5 个部分）
- **内容**：部分列表（可选图标/页码）

### 布局选项

**编号垂直列表** - 最适合 3-5 个部分，简单的演示```
|  TABLE OF CONTENTS            |
|                                |
|  01  Section Title One         |
|  02  Section Title Two         |
|  03  Section Title Three       |
```**两列网格** — 最适合 4-6 个部分、内容丰富的演示```
|  TABLE OF CONTENTS              |
|                                  |
|  01  Section One   02  Section Two  |
|      Description       Description  |
|  03  Section Three 04  Section Four |
```**侧边栏导航** — 最适合 3-5 个部分，现代/企业```
| ▌01 |  Section Title One           |
| ▌02 |  Section Title Two           |
| ▌03 |  Section Title Three         |
```**基于卡片** — 最适合 3-4 个部分，创意/现代```
|  TABLE OF CONTENTS                    |
|  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐  |
|  │ 01  │  │ 02  │  │ 03  │  │ 04  │  |
|  │Title│  │Title│  │Title│  │Title│  |
|  └─────┘  └─────┘  └─────┘  └─────┘  |
```### 字体大小层次结构

|元素|推荐尺寸 |与基础的比率|
|--------|-----------------|------------|
|页面标题（“目录”/“议程”）| 36-44 像素 | 2.5 倍-3 倍 |
|节号 | 28-36 像素 | 2x-2.5x |
|章节标题 | 20-28 像素 | 1.5x-2x|
|部分描述 | 14-16 像素 | 1x（基础）|

**关键原则：**
1. **清晰的编号**：部分编号应在视觉上突出 - 粗体、强调色或更大的尺寸
2. **可扫描结构**：查看者应在 2-3 秒内扫描所有部分
3. **一致的间距**：部分之间的垂直间距相等
4. **视觉标记**：用于锚定每个部分的彩色点、线、数字或图标
5. **避免混乱**：描述最多一行或完全省略

### 内容元素

1. **页面标题** — 始终为必填项（“目录”、“议程”、“概述”）
2. **章节编号** — 一致的格式（01、02...或 I、II...）
3. **章节标题** — 清晰简洁
4. **部分描述** — 可选的一行摘要
5. **视觉分隔符** - SVG 分隔符或间距
6. **装饰元素** - 微妙的强调形状
7. **页码徽章** — **强制**

### 设计决策

1. **节数**：3 → 垂直列表； 4-6 → 网格或紧凑； 7+ → 多列
2. **描述长度**：长→垂直列表；无 → 紧凑网格/卡片
3. **语气**：公司→编号列表；创意→卡片为主；学术 → 罗马数字
4. **一致性**：匹配封面的视觉风格

### 工作流程

1. **分析**：节列表、计数、演示上下文
2. **选择布局**：基于部分计数和内容
3. **规划视觉层次结构**：编号样式、字体大小、间距
4. **编写幻灯片**：使用 PptxGenJS。使用形状作为装饰元素。 **必须包含页码徽章。**
5. **验证**：生成预览，使用 markitdown 提取文本，验证内容和徽章。

## 3. 部分分隔符

- **用途**：主要部分之间的清晰过渡
- **内容**：节号 + 标题（+ 可选的 1-2 行介绍）

### 布局选项

**Bold Center** — 最适合简约、现代的演示```
|                  02                    |
|           SECTION TITLE               |
|         Optional intro line           |
```**与强调块左对齐** — 最适合企业、结构化演示```
| ████ |  02                            |
| ████ |  SECTION TITLE                 |
| ████ |  Optional intro line           |
```**分割背景** — 最适合高对比度、戏剧性的过渡```
| ██████████ |     SECTION TITLE        |
| ██  02  ██ |     Optional intro       |
| ██████████ |                          |
```**带有叠加层的全出血背景** — 最适合创意、大胆的演示```
| ████████████████████████████████████  |
| ████       large 02        █████████ |
| ████    SECTION TITLE      █████████ |
| ████████████████████████████████████  |
```### 字体大小层次结构

|元素|推荐尺寸 |笔记|
|--------|-----------------|--------|
|节号 | 72-120 像素 |粗体、强调色或半透明|
|章节标题 | 36-48 像素 |粗体、清晰、主要文本颜色 |
|简介文字| 16-20 像素 |重量轻，颜色柔和，可选 |

**关键原则：**
1. **戏剧性数字**：章节编号 = 最突出的视觉元素
2. **强标题**：大但明显次于数字
3. **最少内容**：只有编号 + 标题 + 可选的一行字
4. **呼吸空间**：留出充足的空白——分隔线是暂停的时刻

### 内容元素

1. **章节编号** — 始终为必填项。格式：`01`、`02`... 或 `I`、`II`... 匹配 TOC 样式。
2. **章节标题** — 始终为必填项。清晰、简洁。
3. **介绍文本** — 可选的 1-2 行描述。
4. **装饰元素** — SVG 强调形状（条形、线条、几何块）。
5. **页码徽章** — **强制**。

### 设计决策

1. **语气**：公司→重音块；创意→全出血；最小 → 粗体中心
2. **颜色**：背景/强调色的强烈调色板颜色；高对比度文本
3. **一致性**：一个演示文稿中所有分隔线的分隔线样式相同
4. **与内容幻灯片对比**：视觉上不同（不同的背景颜色，更多的空白）

### 工作流程

1. **分析**：章节编号、标题、可选简介
2. **选择布局**：基于内容和语气
3. **编写幻灯片**：使用 PptxGenJS。使用形状作为装饰元素。 **必须包含页码徽章。**
4. **验证**：生成预览、提取文本、验证内容和徽章。

## 4. 内容页面

根据内容选择子类型。每张内容幻灯片都属于一个子类型：

### 子类型

**文本** — 项目符号、引言或短段落
- 仍必须包含图标或 SVG 形状 - 绝不能仅包含纯文本```
|  SLIDE TITLE                          |
|  * Bullet point one                   |
|  * Bullet point two                   |
|  * Bullet point three                 |
```**混合媒体** — 两列或半出血图像 + 文本```
|  SLIDE TITLE                          |
|  Text content     |  [Image/Visual]   |
|  and bullets      |                   |
```**数据可视化** — 图表（SVG 条形/进度/环）+要点
- 必须包含数据源```
|  SLIDE TITLE                          |
|  [SVG Chart]      |  Key Takeaway 1   |
|                   |  Key Takeaway 2   |
|                   Source: xxx          |
```**比较** — 并排的列或卡片（A 与 B，优点/缺点）```
|  SLIDE TITLE                          |
|  ┌─ Option A ─┐  ┌─ Option B ─┐      |
|  │  Detail 1  │  │  Detail 1  │      |
|  └────────────┘  └────────────┘      |
```**时间线/流程** — 带箭头的步骤、旅程、阶段```
|  SLIDE TITLE                          |
|  [1] ──→ [2] ──→ [3] ──→ [4]         |
|  Step    Step    Step    Step          |
```**图像展示** — 英雄图像、图库、视觉优先布局```
|  SLIDE TITLE                          |
|  ┌────────────────────────────────┐   |
|  │         [Hero Image]           │   |
|  └────────────────────────────────┘   |
|  Caption or supporting text           |
```### 字体大小层次结构

|元素|推荐尺寸 |笔记|
|--------|-----------------|--------|
|幻灯片标题 | 36-44 像素 |幻灯片顶部粗体|
|节标题| 20-24 像素 |粗体，用于幻灯片中的子部分 |
|正文 | 14-16 像素 |常规重量，左对齐 |
|字幕/来源| 10-12 像素 |柔和的颜色，最小的文字 |
|统计标注 | 60-72 像素 |关键统计数据的大粗体数字 |

**关键原则：**
1. **左对齐正文** — 切勿将段落或项目符号列表居中
2. **大小对比** — 标题必须为 36pt+，才能从 14-16pt 的正文中脱颖而出
3. **需要视觉元素** — 每张内容幻灯片必须至少有一个非文本元素
4. **呼吸空间** — 0.5" 最小边距，内容块之间 0.3-0.5"

### 内容元素

1. **幻灯片标题** — 始终必需，位于幻灯片顶部
2. **正文内容** — 文本、项目符号、数据或基于子类型的比较
3. **视觉元素** - 图像、图表、图标或 SVG 形状 - 始终必需
4. **来源/标题** — 显示数据或外部内容时
5. **页码徽章** — **强制**

### 设计决策

1. **子类型**：首先确定——驱动整个布局
2. **内容量**：密集→多栏或更小的字体；浅色 → 具有更多空白的较大元素
3. **数据与叙述**：数据密集→图表+统计标注；故事驱动 → 图片 + 引言
4. **多样性**：每张内容幻灯片都应使用与前一张不同的布局
5. **一致性**：版式、颜色和间距必须与演示文稿的其余部分相匹配

### 工作流程

1. **分析**：内容，确定子类型，规划布局
2. **选择布局**：最适合子类型和内容量
3. **编写幻灯片**：使用 PptxGenJS。使用图表、装饰元素、图标的形状。 **必须包含页码徽章。**
4. **验证**：生成预览为 `slide-XX-preview.pptx`。使用 markitdown 提取文本，验证所有内容，无占位符文本，包括徽章。

## 5. 摘要/结束页

- **用于**：总结+行动
- **内容**：要点、CTA/后续步骤、联系方式/QR、谢谢

### 布局选项

**关键要点** — 最适合教育、企业、数据驱动的演示```
|  KEY TAKEAWAYS                        |
|  ✓  Takeaway one                      |
|  ✓  Takeaway two                      |
|  ✓  Takeaway three                    |
```**CTA / 后续步骤** — 最适合销售推介、提案、项目启动```
|  NEXT STEPS                           |
|  [1] Action item one                  |
|  [2] Action item two                  |
|  Contact: email@example.com           |
```**谢谢/联系** — 最适合会议演讲、主题演讲```
|            THANK YOU                   |
|         name@company.com              |
|         @handle | website.com         |
```**分割回顾** — 最适合需要回顾和行动的演示```
|  SUMMARY            |  NEXT STEPS      |
|  * Point one        |  Contact us at   |
|  * Point two        |  email@co.com    |
|  * Point three      |  [QR Code]       |
```### 字体大小层次结构

|元素|推荐尺寸 |笔记|
|--------|-----------------|--------|
|结束标题（“谢谢”/“总结”）| 48-72 像素 |大胆、指挥|
|外卖/行动项目| 18-24 像素 |清晰、可扫描 |
|支持文字| 14-16 像素 |正常体重|
|联系方式| 14-16 像素 |柔和的色彩 |

**关键原则：**
1. **强有力的结束语**：主要信息应该是最大、最突出的
2. **可扫描项目**：简洁的要点/行动项目（每行）
3. **触点清晰度**：清晰但不占主导地位
4. **令人难忘的结局**：自信、完美的结局

### 内容元素

1. **结束标题** — 始终必填
2. **要点** — 3-5 个简洁的总结点（如果适用）
3. **号召性用语** — 明确后续步骤（如果适用）
4. **联系信息** — 电子邮件、网站、社交账号（如果提供）
5. **装饰元素** — SVG 强调视觉一致性
6. **页码徽章** — **强制**

### 设计决策

1. **结束类型**：回顾、号召性用语、致谢或组合
2. **内容量**：诸多要点→罗列；简单的结束 → 居中的感谢
3. **观众行动**：观众需要做某事→CTA；信息 → 要点
4. **色调一致性**：匹配封面的能量
5. **视觉区别**：特殊但与其他部分没有脱节

### 工作流程

1. **分析**：结束内容——要点、CTA、联系方式、感谢
2. **选择布局**：基于内容类型
3. **编写幻灯片**：使用 PptxGenJS。使用形状作为装饰元素。 **必须包含页码徽章。**
4. **验证**：生成预览、提取文本、验证内容和徽章。

## 附加布局模式

在内容幻灯片中使用这些内容以获得视觉多样性：

- **两栏**（左侧为文字，右侧为插图）
- **图标 + 文本行**（彩色圆圈中的图标，粗体标题，下面的说明）
- **2x2 或 2x3 网格**（图像在一侧，内容块网格在另一侧）
- **半出血图像**（完整的左侧或右侧）与内容覆盖
- **大统计标注**（大数字 60-72pt，下面带有小标签）
- **比较栏**（之前/之后，优点/缺点）
- **时间线或流程**（编号步骤、箭头）
- **小彩色圆圈中的图标**位于节标题旁边
- **斜体重音文本**用于关键统计数据或标语
